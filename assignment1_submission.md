## Explanation: Why use a branch + pull request workflow?

Working in a branchâ€“pull request workflow is useful because it keeps the `main` branch more stable: you can experiment and make changes in a separate branch without immediately affecting the production-ready code. Pull requests also create a clear place for code review and discussion, so teammates (or your future self) can check, comment on, and approve changes before they are merged. This workflow supports collaboration and helps catch mistakes early, which reduces the risk of breaking the main branch.
